## ClawGuard Security Policy / 安全策略

### Supported Versions / 支持版本

ClawGuard 仍处于早期阶段（0.x），我们尽量对最新发布版本提供安全修复。  
We are in an early stage (0.x). We aim to provide security fixes for the latest released version.

---

### Reporting a Vulnerability / 报告安全漏洞

**English**

If you believe you have found a security vulnerability in ClawGuard:

1. Please **do not** open a public GitHub issue with exploit details.
2. Instead, contact the maintainer privately (for example via the security contact listed in the repository or project page).
3. Include:
   - A clear description of the issue.
   - Steps to reproduce (PoC).
   - Affected version / commit.
4. We will:
   - Acknowledge your report as soon as possible.
   - Work with you to verify and understand the impact.
   - Prepare a fix and coordinate a responsible disclosure timeline if necessary.

**中文**

如果你认为在 ClawGuard 中发现了安全漏洞：

1. **不要**在 GitHub Issue 中公开利用细节。
2. 请通过项目主页或仓库中列出的安全联系渠道，**私下**联系维护者。
3. 邮件/信息中建议包含：
   - 问题的简要描述。
   - 复现步骤或 PoC。
   - 受影响的版本或 commit。
4. 我们会：
   - 尽快确认收到你的报告。
   - 与你一起验证问题和评估影响范围。
   - 在必要时准备修复并协调合理的披露时间表。

---

### Scope of Responsibility / 责任范围

- ClawGuard 仅对自身代码与发布产物负责；OpenClaw 本身及第三方 Skills 的漏洞不在本项目的直接修复范围内。  
- ClawGuard is responsible only for vulnerabilities in its own code and released artifacts; vulnerabilities in OpenClaw itself or third-party Skills are out of direct scope.
